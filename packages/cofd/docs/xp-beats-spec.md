# XP & Beats Subsystem Specification — Chronicles of Darkness 2e

This document specifies the Beats / Experience economy for the UrsaMU CoFD plugin. All rules are drawn from the **Chronicles of Darkness 2nd Edition** core book (the "God-Machine Chronicle / CofD Core" line). Template-specific rules (Vampire: The Requiem 2e, Werewolf: The Forsaken 2e, Mage: The Awakening 2e, etc.) inherit the same skeleton; differences are noted as "Arcane" where appropriate.

---

## 1. Beats — Earning Sources

Beats are the atomic unit of advancement. A character earns Beats during play; **5 Beats = 1 Experience**. The Storyteller (ST) tracks the master ledger, but players are encouraged to call out Beat triggers when they occur.

### 1.1 Player-Triggered Beats

These are mechanically triggered by player choice or by an event the player initiates. They are awarded **automatically** when the trigger fires — no ST approval required, though the ST should confirm logging.

| Trigger | Beats | Notes |
|---|---|---|
| **Resolve a Condition** | 1 | When a Condition's resolution clause fires, the player marks the Beat and clears the Condition. Persistent Conditions can resolve repeatedly across sessions per their text. |
| **Fulfill an Aspiration** | 1 | Each character maintains 1–3 short/long-term Aspirations. Fulfilling one grants 1 Beat. The player then writes a new Aspiration to replace it. |
| **Exceptional Success on a roll tied to an Aspiration** | +1 (Beat for the exceptional, plus Inspired Condition which resolves for another Beat) | Standard 5+ successes still grants the Inspired Condition, which is itself worth 1 Beat on resolution. |
| **Turn a Failure into a Dramatic Failure** | 1 | The player may elect, *after* seeing the failed roll, to escalate to a Dramatic Failure. They take 1 Beat immediately and the ST narrates worsened consequences. |
| **Take a Beat from a personal Touchstone / Anchor moment** | 1 | Template-specific (e.g., Vampire Touchstones, Mage Obsessions, Werewolf Touchstone). Triggered when the character meaningfully engages or threatens the anchor in a scene. |

### 1.2 ST-Awarded Beats

These require ST adjudication. The ST should aim to keep the table roughly even, awarding 2–4 Beats per character per session as a baseline pace.

| Trigger | Beats | Notes |
|---|---|---|
| **Scene-end / session-end story Beat** | 1–2 | Awarded for meaningful contribution to the unfolding story; one of the most common ST awards. |
| **Risk / sacrifice / dramatic moment** | 1 | When a character takes a notable in-fiction risk (physical, social, or moral) that costs them. |
| **Roleplaying Virtue / Vice (or template equivalent)** | 1 | Mortals: meaningfully invoking Virtue or Vice such that it complicates the scene. Supernaturals use their template's equivalents (Mask/Dirge for Vampires, Blood/Bone for Werewolves, etc.). |
| **Group/story milestone** | 1 | All PCs receive a Beat when a chapter/story arc concludes. |
| **Breaking Point / Integrity hit (and survived)** | 1 | When the character endures a Breaking Point roll, regardless of outcome — they took on narrative weight. |

### 1.3 Arcane Beats (template-specific)

Supernatural templates earn a parallel **Arcane Beat** track, which converts at the same 5:1 ratio into **Arcane Experiences**. Mortals do **not** have Arcane Experiences.

- **Vampire (Requiem 2e)**: Arcane Beats from frenzy, feeding-related Conditions, supernatural Touchstone events, and Predatory Aura usage with consequence.
- **Werewolf (Forsaken 2e)**: Arcane Beats from spirit interactions, Harmony shifts, and lodge/totem obligations.
- **Mage (Awakening 2e)**: Arcane Beats from Mysteries solved, Obsession resolution, Paradox, and Wisdom-relevant choices.
- **Generic rule**: Any Beat earned for a trigger that is "supernatural in nature" should be flagged Arcane by the ST for templates that have such a track.

## 2. Beats → Experience Conversion

- **5 Beats = 1 Experience (XP)**
- **5 Arcane Beats = 1 Arcane Experience (AXP)** — supernaturals only
- Conversion is automatic at the moment the 5th Beat is logged. Excess Beats roll over (e.g., 7 Beats becomes 1 XP + 2 Beats banked).
- Beats and Arcane Beats are tracked **separately** and do not combine. A character cannot spend 3 Beats + 2 Arcane Beats to make 1 Experience.

## 3. Experience Cost Table

The following table reflects the **CofD 2e core** costs. Template books may layer additional traits onto these rows but the costs are stable across the line.

| Trait | Experience Cost per Dot | Notes |
|---|---|---|
| **Attribute** | 4 XP per dot | Buys current rating → next rating (e.g., 2 → 3 costs 4). Non-Arcane. |
| **Skill** | 2 XP per dot | Same per-dot pricing. Non-Arcane. |
| **Skill Specialty** | 1 XP flat | Per specialty; no scaling cost. Non-Arcane. |
| **Merit** | 1 XP per dot | Pay for each new dot. Merit prerequisites must still be met. Non-Arcane. |
| **Willpower (permanent dot)** | 2 XP per dot | Maxes at Resolve + Composure. Non-Arcane. |
| **Integrity / Humanity / Clarity / Wisdom / Harmony** | 2 XP per dot (restoration only) | Used to **restore** lost dots on the template's morality track. Cannot exceed prior maximum without ST approval / breaking point reversal. Non-Arcane. |
| **Supernatural Power** (Discipline, Arcanum, Contract, Renown, Gift, etc.) | **Arcane** XP — typically 4 AXP per dot | Template-defined. E.g., Mage Arcanum, Vampire Discipline, Werewolf Renown, Changeling Contract. |
| **Power Stat** (Blood Potency, Gnosis, Wyrd, Primal Urge) | **Arcane** XP — 5 AXP per dot | Template-defined ceiling (typically 10, gated by other prerequisites). |
| **Mortal → Template upgrade** | Variable (commonly 25–50 XP equivalent, or a story-justified transition) | The 2e core treats this as primarily a narrative event; if mechanized, charge the equivalent of a template's starting supernatural budget. ST gate required. |

### 3.1 Notes on the Table

- "Per dot" means the cost to buy the **next** dot (going from rating *n* to *n+1*). 2e abolished the older "current rating × multiplier" New World of Darkness 1e formula.
- "Arcane" indicates the cost must be paid from the Arcane Experience pool, **not** the standard XP pool.
- Specialty cost is **flat**, not per-dot.
- Lost dots of morality (Integrity, etc.) can be regained for 2 XP per dot, representing therapy / reflection / atonement during downtime.

## 4. Spending Rules

- **When**: XP and AXP are spent during **downtime** — between sessions, or during in-character downtime scenes explicitly framed by the ST.
- **ST Approval Threshold**: Spending Beats (i.e., conversion) is automatic, but **spending Experiences** requires the ST to confirm:
  1. The character had narrative opportunity to learn / grow into the new trait (training, mentor, study, awakening, etc.).
  2. Prerequisites are met (Merit prereqs, Attribute minimums for Skills, template gating for powers).
  3. Power Stat increases additionally require story justification (a major triumph, diablerie, initiation, etc.).
- **No "banking forever"**: There is no cap on stored XP, but the core book advises STs to nudge hoarders to spend; XP represents lived experience and should circulate.
- **"Experiences must be earned, not given"** (CofD 2e core guidance): The ST should not hand out Experiences as bribes, apologies, or session attendance prizes. All XP flows through the Beat economy. If the ST wishes to reward something out-of-band, they grant Beats — not raw Experiences.
- **Mortal → supernatural transition**: When a character changes template, unspent XP and AXP convert per the template book's guidance (often AXP is wiped and refunded as starting supernatural dots; standard XP carries over).

## 5. Proposed Data Shape

The plugin should drive costs from a JSON resource so template plugins can override entries without code changes. Proposed file: `resources/xp_costs.json`.

```json
{
  "version": "cofd-2e-core",
  "conversion": {
    "beatsPerExperience": 5,
    "arcaneBeatsPerArcaneExperience": 5
  },
  "costs": {
    "attribute":           { "costPerDot": 4, "arcane": false },
    "skill":               { "costPerDot": 2, "arcane": false },
    "specialty":           { "costFlat":   1, "arcane": false },
    "merit":               { "costPerDot": 1, "arcane": false },
    "willpower":           { "costPerDot": 2, "arcane": false },
    "integrity":           { "costPerDot": 2, "arcane": false, "restorationOnly": true },
    "supernaturalPower":   { "costPerDot": 4, "arcane": true,  "templateGated": true },
    "powerStat":           { "costPerDot": 5, "arcane": true,  "templateGated": true, "requiresStoryApproval": true },
    "templateUpgrade":     { "costFlat":  25, "arcane": false, "requiresStoryApproval": true, "notes": "ST-adjudicated; varies by target template" }
  },
  "beatSources": {
    "playerTriggered": [
      { "id": "resolve_condition",       "beats": 1, "arcane": false },
      { "id": "fulfill_aspiration",      "beats": 1, "arcane": false },
      { "id": "dramatic_failure_optin",  "beats": 1, "arcane": false },
      { "id": "touchstone_engagement",   "beats": 1, "arcane": "template" }
    ],
    "stAwarded": [
      { "id": "scene_end",               "beats": 1, "arcane": false },
      { "id": "session_end",             "beats": 2, "arcane": false },
      { "id": "risk_sacrifice",          "beats": 1, "arcane": false },
      { "id": "virtue_vice_roleplay",    "beats": 1, "arcane": false },
      { "id": "story_milestone",         "beats": 1, "arcane": false },
      { "id": "breaking_point",          "beats": 1, "arcane": false }
    ]
  },
  "rules": {
    "experiencesEarnedNotGiven": true,
    "mortalsHaveArcaneExperience": false,
    "arcaneAndStandardArePoolsSeparate": true,
    "spendingRequiresDowntime": true,

```

### 5.1 Notes for Implementers

- The `arcane` field on a beat source may be the literal `false`, `true`, or the string `"template"` — meaning the template plugin decides whether this trigger goes to the Arcane track.
- `templateGated: true` on a cost entry means the trait list (which Disciplines, which Arcana, etc.) is supplied by the template plugin, not by core.
- The plugin should expose `+xp/award`, `+xp/spend`, `+beats/add`, `+beats/list` commands, all routed through `u.db.modify` against the character's `state.cofd.xp` substate. Conversion 5→1 must happen on every Beat write.
- Per the project's command invariants, all writes must be guarded by `await u.canEdit(u.me, target)` and `isAdmin` for ST-awarded events.
